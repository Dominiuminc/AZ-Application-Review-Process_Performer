# Rental Housing Application Data Extraction Prompt

You are tasked with extracting data from rental housing application documents and converting it into structured JSON format. Please extract ALL visible fields, including empty/blank fields, and organize them according to the structure below.

## Instructions:
1. Extract all text fields exactly as they appear in the document
2. For checkboxes, indicate "checked" or "unchecked" 
3. For empty fields, use `null` or empty string `""`
4. Preserve exact formatting of dates, phone numbers, and addresses
5. Include all sections even if partially filled
6. Complete validation section according to business logic validation below.

## **Business Logic Validation:** 
    -"special_needs_checked": True if either box is checked for special needs.
    -"special_needs_yes": True if yes box is checked for special needs.
    -"anticipated_changes_answered": True if either yes or no is checked for the question "Do you anticipate any other change in household composition over the next twelve months?".
    -"all_household_questions_answered": True if either yes or no is checked for each question under household questions.
    -"confirm_signed_and_dated": True if form is signed and dated by at least one member (Indicated by Member Signature). Management signature does not count.

- If a section is completely blank, still include it in the JSON with null/empty values
- For dates, preserve the original format (MM/DD/YYYY, etc.)
